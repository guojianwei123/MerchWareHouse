# UE/UI 设计方案：移动端收纳应用化

## 目标

当前 Web 入口只是功能验证壳，视觉和交互更接近后台表单。后续 UI 重构的目标是把“谷子仓库”做成移动端收纳应用：用户第一眼看到的是自己的收藏空间、物品和资产，而不是页面按钮和裸表单。

参考竞品位于 `docs/competing-products/心水收纳/`。可借鉴的方向是：柔和底色、手账卡片、底部导航、贴纸感图标、资产挂牌、半屏录入面板和房间/收纳空间叙事。

## 产品框架

### 主导航

使用底部导航替代顶部横排按钮。导航项固定为：

- `房间`：展示柜、房间和收纳空间。
- `物品`：库存列表、搜索、分类筛选。
- `录入`：中间凸起主入口，打开录入面板。
- `品类`：分类管理，使用收纳册/文件夹视觉。
- `我的`：资产总览、导出、提醒、设置、风格。

桌面端不改信息架构，只把移动端外壳居中展示成手机画布；不要退回后台管理布局。

### 录入入口

`录入` 是主动作，不做普通 tab 等权处理。点击后打开录入面板：

- `AI 录入`：主入口，识别小票、订单、谷子图片。
- `手动录入`：用户自行填写。
- `链接解析`：粘贴商品链接。
- `批量导入`：多图或表格导入。

URL 输入仅作为开发/高级输入，默认折叠，不进入第一视觉层级。

## 页面设计

### UploadPage

页面标题：`今天收了什么谷？`

结构：

1. 情绪顶部区：柔和插画感背景、简短说明。
2. 主功能卡：`AI 录入`，显示识别状态、剩余次数、当前模型。
3. 高级输入折叠区：图片 URL 输入。
4. 功能三宫格：手动录入、链接解析、批量导入。

成功识别后跳转/切换到草稿确认，不在上传页展示完整 JSON。

### DraftReviewPage

使用“确认小票”视觉。目标是让用户快速确认和修正，而不是阅读 JSON。

结构：

1. 图片预览和分类标签。
2. 识别结果卡：名称、IP、角色、系列。
3. 价格区：官方价、购入价、市价。
4. 分类字段区：
   - 吧唧：直径、形状。
   - 纸片：长、宽、材质。
   - 亚克力：高度、宽度、底座。
   - 其他类型按 schema 渲染核心字段。
5. 底部固定操作：确认入库、重新识别、放弃草稿。

### RoomEditorPage

设计为“类 3D 展示柜”，不再显示普通矩形编辑框。

视觉结构：

- 柜体：背板、左右侧板、层板、格子阴影。
- 透视：柜体顶部略窄、侧板略深，表达空间深度。
- 物品：按类型渲染成贴纸、徽章、亚克力立牌等卡片。
- 状态牌：柜顶显示标题和公开/私密状态。
- 工具条：添加层板、添加物品、保存、分享。

交互规则：

- 拖拽时物品保持贴纸阴影。
- 越界或重叠时使用柔和红色描边和气泡提示。
- 保存按钮在存在非法节点时禁用。
- 分享入口只在布局有效时可用。

实现阶段可以先用 CSS 2.5D：`perspective`、`transform`、多层阴影和伪元素即可；不强制引入 Three.js。

### AssetDashboardPage

使用“资产挂牌”视觉，避免表格。

结构：

1. 顶部挂牌：资产总值、日期。
2. 小指标：总投入、当前估值、物品数量、盈亏。
3. 分组统计卡：IP、角色、系列、分类。
4. 空状态：`还没有入库的谷子`，提供去录入入口。

分组项显示金额、数量、占比，不使用 HTML 表格作为主视觉。

### ShowcaseSharePage

只读公开展示页，强调“这是一个公开展示柜”。

结构：

1. 展示柜标题和公开说明。
2. 类 3D 展示柜预览。
3. 公开物品列表。
4. 底部操作：保存同款布局、返回。

隐私规则：

- 只展示名称、IP、角色、系列、分类、图片。
- 禁止展示官方价、购入价、市价、价格记录、内部详情。

### Category Page

后续新增 `CategoryPage`，用于替代当前无页面承载的“品类”导航。

视觉使用收纳册/文件夹：

- 吧唧
- 纸片
- 亚克力
- 立牌
- 手办
- 布艺
- 特殊

每个分类卡显示启用状态、数量和最近新增。

### Profile Page

后续新增 `ProfilePage`，承载“我的”导航。

模块：

- 用户同步状态。
- 资产摘要。
- 数据导出。
- 提醒通知。
- 主题风格。
- AI 生成风格入口，生成结果仅应用主题 token。

## 视觉系统

默认风格是 `aqua-opera`。整体风格应该是珍珠白、水蓝、白金描边和剧场感，但不能牺牲信息可读性。该主题只抽象水纹、舞台、玻璃柜和优雅边框，不使用任何受保护角色、Logo 或官方素材。

原则：

- 不使用纯黑正文，使用深棕。
- 主卡圆角 18-28px。
- 功能卡可以有贴纸感描边，但不要过多装饰。
- 页面背景可有轻微纸纹，透明度必须低。
- 图标优先使用线性图标或贴纸风图标，按钮内不使用冗长说明。
- 移动端优先，桌面端展示为居中手机画布。

## 主题 Token

主题由 CSS 变量驱动。组件只读取 token，不硬编码主题色。

必备 token：

- `--color-bg`
- `--color-surface`
- `--color-surface-soft`
- `--color-primary`
- `--color-primary-strong`
- `--color-text`
- `--color-muted`
- `--color-line`
- `--radius-card`
- `--radius-pill`
- `--shadow-card`
- `--texture-opacity`

预设主题：

- `aqua-opera`：水蓝剧场，默认主题。
- `cream-desk`：奶油手账，可选主题。
- `mint-cabinet`：薄荷收纳柜，偏清爽。
- `pink-ticket`：粉色小票/贴纸，偏可爱。

后续 AI 生成风格只允许输出 token 包，不直接生成业务组件或改动页面结构。

## 验收标准

- 第一屏不再像后台管理页，移动端观感接近收纳类 App。
- 所有主页面都有顶部情绪区、中部内容区、底部导航或操作区。
- 展示柜能表达柜体深度、层板、格子和物品层次。
- 主题可通过 token 替换，不需要改组件结构。
- UI 重构不得破坏现有 Zod 校验、内存仓储、测试和业务服务。
