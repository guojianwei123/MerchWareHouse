# 计划 5：资产统计看板 MVP

## 目标
基于 `GuziItem` 和 `PriceRecord` 计算用户收藏资产概览，并在前端展示总投入、当前估值和分组占比。

## 验收标准
- [x] 在 `src/service/` 中实现资产统计服务，输入库存项和价格记录，输出总投入、当前总估值、官方发售价汇总和盈亏估算。
- [x] 支持按 IP、人物、系列、分类聚合购入价和当前估值。
- [x] 前端页面或状态层能读取统计结果并展示空库存状态。
- [x] 统计逻辑对缺失价格字段使用明确默认值，不能产生 `NaN`。

## 边界
- 不自动抓取外部市场价格。
- 不实现复杂图表库集成；MVP 可以使用基础列表或简单占比展示。
- 不修改展示柜分享逻辑。

## 最小验证方式
- 使用 3 个以上库存样例验证总投入、当前估值、官方发售价和盈亏。
- 验证按 IP、人物、系列、分类聚合结果正确。
- 验证空库存返回 0 汇总和空分组。
