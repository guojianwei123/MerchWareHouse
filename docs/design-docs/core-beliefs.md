# 核心设计决策

1. **多维度标签树而非严格层级结构**
   - **为什么**：谷子的分类非常灵活，可能属于多个系列或组合。强制层级（IP->人物->系列）会导致一些商品难以分类。
   - **决策**：使用扁平化的标签系统（Tag System），内部按类型（IP标签、人物标签）区分，多对多关联。

2. **价格历史记录机制**
   - **为什么**：谷子的市场价是波动的（所谓的“海景房”或“跌破原价”）。
   - **决策**：价格（特别是市价）作为时序数据存储，保留历史快照，方便计算涨跌幅。

3. **移动端收纳应用优先，而非后台管理工具**
   - **为什么**：目标用户是在整理个人收藏，核心情绪是“收纳、展示、记录”，不是处理企业后台数据。
   - **决策**：主体验采用底部导航、手账卡片、贴纸感信息层级和类 3D 展示柜。Web 端也按移动端应用壳居中呈现，不回退到后台表格布局。

4. **主题风格由 Token 驱动**
   - **为什么**：用户后续可能希望用 AI 生成自己的风格，不能让风格耦合到业务组件。
   - **决策**：颜色、圆角、阴影和纹理由主题 token 控制。AI 生成风格只输出 token 包，不直接生成业务组件或改动页面结构。
